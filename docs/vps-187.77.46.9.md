# Aplicação na VPS – 187.77.46.9

## URLs

| Uso            | URL |
|----------------|-----|
| **Frontend (site)** | http://187.77.46.9 |
| **Backend (API)**   | http://187.77.46.9:8080 |
| **Swagger**         | http://187.77.46.9:8080/swagger-ui/index.html |

O frontend chama a API em **`/api`** (ex.: `http://187.77.46.9/api/v1/auth/login`). O Nginx no container do frontend faz proxy de `/api` para o backend na porta 8080.

---

## Checklist na VPS

1. **Firewall** – liberar portas 80 (frontend) e 8080 (backend):
   ```bash
   ufw allow 80
   ufw allow 8080
   ufw reload
   ```

2. **Backend** – rodando e acessível em `http://187.77.46.9:8080` (ex.: `java -jar app.jar` ou systemd).

3. **Frontend** – container Nginx na porta 80:
   ```bash
   cd /home/ubuntu/dev/frontend
   docker compose up -d --build
   ```

4. **Postgres** – se usar Docker:
   ```bash
   cd /home/ubuntu/dev
   docker compose -f docker-compose.db.yml up -d
   ```
